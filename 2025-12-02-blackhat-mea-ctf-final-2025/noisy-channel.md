The server generates 20000 random numbers, and we can select some of them so that they xor to zero. Python random uses MT19937, so every bit generated by it, is a linear combination of `32 * 624`-bit internal state in `GF(2)` arithmetic. Since we cannot control the internal state, we hope that, give a linear combination of some `noise`, they sum to zero regardless of the value of the internal state. That is, we need to find a subset of the `noise` that, always xor to zero, no matter what is the internal state.

So, how to find it? First, we need to encode the `20000*136`-bit generated data as linear combinations of `32 * 624`-bit internal state. We regard each generated bit as a vector, of the coefficients for the `32 * 624`-bit internal state. We want to find a linear combination of these vectors, so that all elements become zero. This is essentially constructing a matrix space and finding its kernel (null space). Let's define the process in math:

Let the unknown `32 * 624 = 19968`-bit internal state of MT19937 named $s_1, ..., s_{19968}$, so every generated bit $g_i$ is a linear combination of these unknown bits in `GF(2)` arithmetic:

$$
g_i = a_{i,1}s_1 + a_{i,2}s_2 + \cdots + a_{i,19968}s_{19968} = \Sigma_{j=1}^{19968}a_{i,j}s_j
$$

Now, consider the LSB bit of 20000 generated 136-bit numbers. So we have 20000 $g_i$. We need to find a linear combination of these $g_i$ so that they always equal to zero, regardless of $s_i$:

$$
b_1g_1 + b_2g_2 + \cdots + b_{20000}g_{20000} = \Sigma_{i=1}^{20000} b_ig_i\\
= \Sigma_{i=1}^{20000} b_i(\Sigma_{j=1}^{19968}a_{i,j}s_j) \\
= 0
$$

All coefficients of `s_j` should equal to zero. So, we can express the equation into matrix:

$$
M = \begin{pmatrix}
a_{1,1} & a_{2,1} & \cdots & a_{20000,1} \\
a_{1,2} & a_{2,2} & \cdots & a_{20000,2} \\
\vdots  & \vdots  & \ddots & \vdots \\
a_{1,19968} & a_{2,19968} & \cdots & a_{20000,19968}
\end{pmatrix}
$$

Vector $b=(b_1, b_2, \cdots, b_{20000})^T$, so:

$$
Mb = (0, 0, \cdots, 0)^T
$$

To solve $b$, we only need to find the kernel (null space) of matrix $M$, and use its basis. If we find such $b$, we can find a linear combination of the `noise` that, the LSB bit of XOR result should always be zero.

```python
from gf2bv import LinearSystem
from gf2bv.crypto.mt import MT19937
from sage.all import *
import tqdm

ls = LinearSystem([32] * 624)
gen = ls.gens()
rng = MT19937(gen)

msg = b"give me the flag!"

values = []
for _ in range(20000):
    values.append(rng.getrandbits(len(msg) * 8))

states = 32 * 624
count = 20000
nbits = 1
M = matrix(GF(2), states * nbits, count)
for i in tqdm.trange(count):
    for j in range(nbits):
        bits = values[i]._bits[j]
        temp = []
        for k in range(states):
            M[k + states * j, i] = bits & 1
            bits >>= 1

space = M.right_kernel()
print(space)
basis = space.basis()[0]
# save it to file later
print(basis)
```

We found a valid basis that no only makes the LSB bit zero, but also accidentally (actually we can add all 136 bits to the equation, but they turned out to have the same solution) makes the whole XOR result zero:

```python
from gf2bv.crypto.mt import MT19937

import random

for test in range(5):
    # randomize internal state
    random.seed()
    test_state = list(random.getstate()[1][:-1])

    # linear combination
    res = 0
    for i in range(20000):
```
